# 字幕与画面知识点时序问题笔记

这份笔记用于记录当前项目里，为什么会出现“字幕和语音是同步的，但字幕比画面中的知识点更早出现”的问题，以及后续应该如何避免。

## 现象

在一些章节里会看到下面这种错位：

- 旁白已经开始讲某个知识点
- 字幕也已经显示这句话
- 但对应的章节标题、bullet、flow 卡片或者 summary 条目还没有完全出现

用户感知上就会变成：

- “字幕比内容早了”

实际上它不一定是字幕单独提前了，也可能是画面内容进场太慢了。

## 这次排查出的三个根因

### 1. 字幕存在全局提前量

之前 `buildSubtitleTrackFromTimedCues()` 在部分 lesson 里使用了 `leadInFrames`，会把字幕开始时间往前挪几帧。

这会带来一个问题：

- 字幕会比真实配音更早出现
- 如果画面知识点本来就有入场动画，错位会更明显

### 2. scene 时长按估算分配，而不是按真实语音分章

早期做法是按一组“节奏权重”去分配各章节时长。

这种方法的问题是：

- 总时长虽然接近旁白总长
- 但每一章的切换点不一定和真实语音句子边界一致
- 于是会出现“上一章还没切走，但下一章的语音和字幕已经开始了”

这也是后来在：

- `prototype 到底是什么？`
- `两个最常见误区`

这类章节里出现错位的核心原因。

### 3. 画面知识点本身有延迟入场动画

`BulletScene`、`FlowScene`、`SummaryScene` 都有逐条 reveal 的逻辑。

如果知识点卡片默认完全不可见，直到十几帧后才慢慢淡入，那么即使字幕和旁白完全同步，用户依然会觉得：

- 字幕先到了
- 知识点还没出现

所以这不是“字幕系统”的单点问题，而是：

- 字幕时间
- 章节切换时间
- 画面 reveal 时间

三者叠加后的结果。

## 这次采用的解决方法

### 1. 取消字幕的提前显示

原则：

- 字幕默认不要早于语音
- 先保证“字幕跟语音同步”
- 再去调整画面内容的入场时机

因此当前修复里，不再给这条 lesson 的字幕增加额外 lead-in。

### 2. 用真实 cue 分割章节时长

不要再只靠“估计每章应该占多少秒”。

更稳的做法是：

1. 先生成真实配音
2. 从配音拿到句子级时间戳
3. 为每个 scene 指定一个“结束 cue 索引”
4. 用 cue 的真实结束时间来计算该 scene 的结束帧

这样做的好处：

- 每一章的切换点会和真实旁白边界对齐
- 不会出现“上一章画面还在，但下一章字幕已经开始”的问题
- 针对单个章节修正时，只需要调对应的 cue 边界

### 3. 让知识点先可见，再逐步强调

对于 bullet / flow / summary 这种场景，当前采用的策略是：

- 不让知识点在入场前完全不可见
- 先以较低透明度显示
- 再通过 opacity 和轻微位移做强化

这样用户在字幕到达时，至少已经能看到对应知识点所属的视觉区域，不会产生“字幕悬空”的感觉。

## 当前推荐规则

后续新视频建议遵守下面几条：

### 规则 1：字幕时间优先跟真实配音对齐

不要手写估算字幕秒数，也不要默认给字幕提早若干帧。

### 规则 2：章节边界优先跟真实 cue 对齐

新 lesson 如果章节很多，优先像这次一样：

- 为每个 scene 显式指定结束 cue

而不是只写一个大致的 seconds。

### 规则 3：知识点场景不要“长时间完全不可见”

尤其是：

- `BulletScene`
- `FlowScene`
- `SummaryScene`

如果用户希望“字幕和知识点一起到”，这些场景最好让知识点容器尽早出现，只把“强调感”做成渐进动画。

### 规则 4：定位问题时先分清是哪一种“早”

排查时先问自己：

1. 是字幕早于语音？
2. 是字幕和语音同步，但章节切换晚了？
3. 还是章节切换对了，但知识点 reveal 太慢了？

这三种情况的修法不同，不要混在一起处理。

## 推荐排查顺序

以后再遇到类似问题，建议按这个顺序排查：

1. 看字幕 cue 的 `startMs / endMs` 是否和真实配音一致
2. 看当前 scene 的开始帧和结束帧是否跨错了章节
3. 看 scene 内部元素是不是在字幕出现后才 reveal
4. 最后再决定是调字幕、调章节边界，还是调 scene 动画

## 一句话总结

“字幕比内容早”通常不是一个点的问题，而是：

- 字幕提前量
- 章节切换边界
- 画面 reveal 动画

三个时间轴没有对齐。

最稳的修法是：

- 用真实配音 cue 驱动字幕
- 用真实 cue 切章节
- 让知识点先可见，再做强调动画
